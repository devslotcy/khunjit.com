1) Ürün Tanımı (Tek Cümle)

Hasta, psikologların müsait slotlarına göre randevu alır, seans öncesi ödeme yapmadan seansa giremez, saat gelince görüntülü görüşme başlar; hasta-psikolog mesajlaşır; admin her şeyi denetler/yönetir (soft delete + audit log ile).

2) Panel Panel Kapsam
A) Hasta (Kullanıcı) Paneli – Net liste

1) Profil & Güvenlik

Kayıt/giriş/şifre sıfırlama

Profil: ad-soyad, iletişim, timezone

“Hesap kapatma” (hard delete değil, soft delete talebi)

2) Psikolog Keşif

Psikolog listeleme + filtre: uzmanlık, dil, ücret aralığı, müsaitlik

Psikolog profil sayfası: bio, uzmanlıklar, diller, ücret, değerlendirme (opsiyonel)

3) Randevu/Booking

Boş slot görüntüleme (psikolog müsaitliklerinden üretilir)

Slot seçince rezervasyon (slot lock): 10 dk ödeme penceresi

Randevu durumları: reserved → payment_pending → confirmed → ready → in_session → completed / cancelled / expired / refunded

4) Ödeme

Randevuda ödeme: ödenmeden seans açılmaz

Profilde Ödeme Geçmişi:

brüt tutar, KDV, KDV hariç, site komisyonu, psikolog neti, ödeme durumu, iade durumu

Fatura/fiş (MVP’de kayıt yeter; PDF sonra)

5) Seans (Görüntülü Görüşme)

Seans saatinden örn. 10 dk önce “Katıl” aktif

Ödeme yoksa buton pasif + “Ödeme bekleniyor”

6) Mesajlaşma

Sadece “ilişkili” psikologlarla (randevusu olan / olmuş) konuşma açılır

Mesaj geçmişi, okunma (opsiyonel)

“Raporla/şikayet” (admin inceleme tetikler)

B) Psikolog Paneli – Görüntülü görüşme odaklı

1) Profil

Bio, uzmanlık, diller, ücret, çalışma prensipleri

Admin doğrulama durumu: verified/unverified

2) Müsaitlik Yönetimi

Haftalık çalışma saatleri (rule)

İstisna günler (exception): izin, tatil, özel slotlar

Slot süresi: 30/45/60 dk gibi

3) Randevular

Yaklaşan randevular listesi (filtre: bugün/hafta)

İptal / yeniden planlama isteği (kural bazlı)

Ödeme durumu görünür: “confirmed değilse seans başlayamaz”

4) Görüntülü Seans

Seans saatinde “Seansı Başlat / Katıl”

Token/room ile güvenli giriş (sadece o randevunun hasta+psikoloğu)

Cihaz testi (kamera/mikrofon seçimi)

Seans bitir → otomatik “completed”

5) Seans Sonrası

Seans notu (varsayılan: sadece psikolog görür)

Seans geçmişi (tarih, süre, durum, kazanç)

6) Kazançlar

Seans bazında: brüt, KDV, site payı, psikolog net

Haftalık/aylık özet

C) Admin Paneli – “Her şeyi kontrol etsin” ama güvenli

1) Kullanıcı Yönetimi

Hasta/psikolog listeleme, arama, status yönetimi (active/blocked)

Soft delete / restore (opsiyonel)

2) Psikolog Doğrulama

Evrak kontrol (MVP: manuel onay checkbox + not alanı)

verified/unverified yönetimi

3) Randevu Yönetimi

Tüm randevular: filtre (tarih, psikolog, hasta, durum)

İptal / yeniden planlama / override

No-show işaretleme

4) Ödeme & İade Yönetimi

Tüm ödemeler listesi, durumlar

İade başlatma (refund_pending → refunded)

Komisyon/KDV ayarları (tek yerden)

5) Mesaj Denetimi (gizlilik korumalı)

Admin normalde mesaj içeriklerini görmez

Sadece “raporlanan konuşmalar” incelemeye açılır

6) Audit Log

Kim neyi ne zaman değiştirdi (özellikle ödeme, randevu, profil, doğrulama)

Geriye dönük denetim için zorunlu

3) “Söylenmesi Gereken” Kritik Kurallar (işin bel kemiği)

Bunlar yazılımda kural olarak geçmeli:

Ödeme olmadan seans yok
appointment.status !== confirmed ise meeting link/token üretilmez.

Double-booking asla olmayacak
Slot seçilince 10 dk rezervasyon + DB transaction/unique constraint.

Soft delete
Hard delete yok. Her kritik tabloda deleted_at, deleted_by.

Audit log zorunlu
Admin/psikolog kritik aksiyonları loglanır.

Timezone standardı
DB’de her şey UTC, ekranda kullanıcı timezone’una çevrilir.

Webhook güvenliği + idempotency (ödeme için)
Webhook iki kez gelse bile aynı ödemeyi iki kere işlemesin.

Mesaj gizliliği
Admin mesajı “rastgele” görmez; sadece rapor/şikayetle açılır.

9) Mobil app bağlanabilirlik (şimdiden doğru kararlar)

Şimdi web yazarken şunları yaparsan mobil entegrasyon çok kolay olur:

API-first: Tüm iş mantığı /api/* altında; UI sadece tüketir.

Versiyonlama: /api/v1/... (ileride kırmadan geliştirirsin)

Auth: mobile için JWT refresh modeli daha rahat; webde cookie session da olur.
İstersen baştan “JWT + refresh token” kurgula (hem web hem mobil ortak).

DTO/Schema: Zod şemalarını hem API hem UI’da ortak kullan (tek kaynak).

CORS: Şimdiden mobile origin’e izin verecek şekilde planla.

Push bildirim: Şimdilik email; sonra mobil push eklenebilir (tasarımda event log tut).

10) Replit Agent’a Vereceğin “MASTER PROMPT v2” (tam derlenmiş)

Bunu kopyala-yapıştır yap. (Web only + mobile-ready)

Türkçe konuş. Next.js (App Router) + TypeScript ile fullstack “Online Görüntülü Psikolog Platformu” yap.
ŞU AN SADECE WEB. Ama ileride mobil app bağlanacağı için API-first tasarla (REST /api/v1).
3 rol: patient, psychologist, admin. Role-based access zorunlu.

ÇEKİRDEK ŞARTLAR:
1) Booking: psikolog müsaitliklerinden slot üretilecek. Double-booking asla olmayacak.
   - Slot seçilince appointment reserved + reserved_until=now+10dk
   - 10dk içinde ödeme olmazsa expired
   - DB transaction + unique constraint ile aynı slot iki kişiye verilmesin
2) Ödeme: Seans başlamadan önce ödeme zorunlu.
   - confirmed değilse “Katıl” linki/token üretilmez
   - ödeme webhook’u idempotent olmalı
3) Muhasebe kırılımı Payment tablosunda ayrı kolonlarla tutulacak:
   gross_amount, vat_rate, vat_amount, net_of_vat,
   platform_fee, platform_vat_amount, processor_fee, provider_payout, currency, status
4) Mesajlaşma: Hasta-psikolog sadece aralarında randevu ilişkisi varsa mesajlaşır.
   Admin mesaj içeriklerini normalde görmez; sadece “raporlanan konuşmalar” açılır.
5) Admin her şeyi yönetir: users, psychologist verify, appointments, payments/refunds, settings(KDV/komisyon), reported chats, audit logs.
6) Hard delete yok: soft delete (deleted_at/deleted_by) + audit log zorunlu.
7) Timezone: DB’de UTC, UI’da kullanıcı timezone’una çevir.
8) Video (MVP): Jitsi embed ile unique room. Sadece confirmed + seans time-window içinde join edilir.
   İleride Daily/Twilio eklenebilir; şimdilik provider interface/stub bırak.

TEKNOLOJİ:
- Next.js App Router + Tailwind
- Prisma ORM + PostgreSQL (ENV DATABASE_URL)
